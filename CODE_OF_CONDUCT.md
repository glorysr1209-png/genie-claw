# Code of Conduct

## Our Standard

GenieClaw is built by people with different backgrounds, skill levels, and
hardware access. We expect everyone participating in this project to help keep
the community practical, respectful, and focused on shipping reliable local AI
software.

Examples of behavior that supports that standard:

- Be direct about technical risks without attacking people.
- Assume good intent, but correct misinformation clearly.
- Give useful review feedback: explain the bug, the impact, and a path forward.
- Respect that not every contributor has Jetson hardware; ask for validation
  gaps to be stated honestly rather than hidden.
- Credit work accurately and avoid misrepresenting authorship, testing, or
  results.

Examples of unacceptable behavior:

- Harassment, threats, stalking, or sustained personal attacks.
- Insults, slurs, sexualized language or imagery, or unwelcome sexual attention.
- Doxxing or publishing private information without explicit permission.
- Trolling, repeated disruption, or knowingly wasting maintainer time.
- Misleading claims about testing, security impact, benchmark results, or
  authorship.

## Scope

This code applies in all GenieClaw project spaces, including GitHub issues,
pull requests, discussions, review comments, project chat, and any public
representations made on behalf of the project.

It also applies when behavior outside project spaces directly affects the
safety or integrity of the GenieClaw community.

## Enforcement

Maintainers may remove, edit, or reject comments, commits, issues, pull
requests, and other contributions that violate this code.

Depending on severity and history, maintainers may also:

- Ask for clarification or a correction.
- Issue a private or public warning.
- Temporarily limit participation.
- Ban a participant from project spaces.
- Escalate serious threats or illegal behavior to the appropriate platform or
  authority.

Enforcement decisions should be proportionate, consistent, and focused on
protecting the project and its contributors.

## Reporting

Report code-of-conduct concerns privately by emailing **<contact@genieclaw.org>**.
Include links, screenshots, or other context when available.

Do not use this channel for vulnerability reports. Security issues should follow
the process in [SECURITY.md](SECURITY.md).

Maintainers will review reports as promptly as practical and will respect the
privacy of reporters and affected people as much as possible.
